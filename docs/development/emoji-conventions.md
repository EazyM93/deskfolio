# 📝 Emoji Conventions

Emojis may be used sparingly in DeskFolio documentation to improve scanning. They should clarify meaning, not decorate text.

Use at most one emoji per heading or status indicator unless there is a strong readability reason.

## General Documentation

| Emoji | Meaning |
|---|---|
| 📌 | Important information |
| 🏗 | Architecture |
| 🧠 | Design rationale or AI memory |
| ⚙ | Configuration |
| 🗄 | Database |
| 🧩 | Components |
| 🎨 | UI |
| 🚀 | Features |
| 📋 | Specifications or workflow |
| 📝 | Notes or conventions |
| ⚠ | Warnings |
| ❌ | Deprecated or removed |
| 💡 | Ideas |
| 🧪 | Testing |
| 📈 | Performance |
| 🔒 | Security |
| 📦 | Dependencies, packaging, releases or changelog |
| 📚 | Documentation |
| 🛠 | Refactoring |
| 🗺 | Roadmap |

|---|---|

## Release Documentation

Use `📦` for changelog and release note headings.

Use these category indicators inside `CHANGELOG.md` and files under `docs/releases/`:

| Emoji | Release Category | Use For |
|---|---|---|
| ✨ | New Feature | New user-facing functionality |
| 🐛 | Bug Fix | Defect fixes and behavioral corrections |
| 🛠 | Improvement | Enhancements to existing behavior, UX or maintainability |
| 🔒 | Security | Security fixes, hardening and privacy-related changes |
| ⚡ | Performance | Speed, memory, startup or responsiveness improvements |
| 🗄 | Database | Schema, migration and persistence changes |
| 🎨 | UI | Visual, layout, theme and interaction changes |
| 🧪 | Testing | Test coverage, test infrastructure and quality checks |
| 📚 | Documentation | Documentation-only changes |
| 📦 | Packaging | Build, distribution and release packaging changes |
| ⚙ | Configuration | Build, runtime or environment configuration changes |
| ❌ | Removed | Removed features, APIs or obsolete behavior |
| ⚠ | Known Issue | Known limitations, regressions or unresolved problems |
| 🔁 | Migration | User-facing upgrade or data migration notes |

Use status indicators for release state:

- `🔵 Planned`
- `🟡 In Progress`
- `✅ Released`
- `🔴 Blocked`

## Rule

If a new emoji convention becomes common, add it here instead of inventing meanings in individual documents.
